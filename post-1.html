Xây Dựng Ứng Dụng Mobile với Flutter

Flutter Mobile Development Dart

Flutter là một framework mã nguồn mở của Google để xây dựng ứng dụng mobile đa nền tảng. Với một codebase duy nhất, bạn có thể tạo ứng dụng cho cả iOS và Android với hiệu suất gần như native.

Tại Sao Chọn Flutter?

Flutter sử dụng Dart programming language và cung cấp một bộ widget phong phú, cho phép developers tạo UI đẹp mắt và responsive một cách nhanh chóng.

Ưu Điểm Chính

  • Hot Reload: Xem thay đổi ngay lập tức mà không cần restart app
  • Single Codebase: Viết một lần, chạy trên cả iOS và Android
  • Performance: Compile thành native code, hiệu suất cao
  • Rich Widgets: Bộ widget phong phú và customizable

Widget Tree và Composition

Flutter sử dụng widget tree để xây dựng UI. Mọi thứ trong Flutter đều là widget:

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Flutter Demo',
      home: Scaffold(
        appBar: AppBar(
          title: Text('Flutter Blog'),
        ),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              Text('Hello Flutter!'),
              ElevatedButton(
                onPressed: () {},
                child: Text('Click Me'),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

State Management

Quản lý state là một phần quan trọng trong Flutter. Có nhiều cách tiếp cận:

setState() - Cho StatefulWidget

class CounterWidget extends StatefulWidget {
  @override
  _CounterWidgetState createState() => _CounterWidgetState();
}

class _CounterWidgetState extends State<CounterWidget> {
  int _counter = 0;
  
  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }
  
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Counter: $_counter'),
        ElevatedButton(
          onPressed: _incrementCounter,
          child: Text('Increment'),
        ),
      ],
    );
  }
}

Provider - State Management Package

Provider là một cách phổ biến để quản lý state trong Flutter:

import 'package:provider/provider.dart';

class CounterProvider extends ChangeNotifier {
  int _count = 0;
  
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// Sử dụng trong widget
Consumer<CounterProvider>(
  builder: (context, counter, child) {
    return Text('Count: ${counter.count}');
  },
)

Navigation và Routing

Flutter cung cấp Navigator để điều hướng giữa các màn hình:

// Navigate to new screen
Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => SecondScreen()),
);

// Navigate back
Navigator.pop(context);

// Named routes
MaterialApp(
  routes: {
    '/': (context) => HomeScreen(),
    '/details': (context) => DetailsScreen(),
  },
)

Async Operations và API Calls

Xử lý các operations bất đồng bộ với Future và async/await:

import 'dart:convert';
import 'package:http/http.dart' as http;

Future<Map<String, dynamic>> fetchData() async {
  final response = await http.get(
    Uri.parse('https://api.example.com/data'),
  );
  
  if (response.statusCode == 200) {
    return json.decode(response.body);
  } else {
    throw Exception('Failed to load data');
  }
}

// Sử dụng trong widget
FutureBuilder<Map<String, dynamic>>(
  future: fetchData(),
  builder: (context, snapshot) {
    if (snapshot.hasData) {
      return Text(snapshot.data!['title']);
    } else if (snapshot.hasError) {
      return Text('Error: ${snapshot.error}');
    }
    return CircularProgressIndicator();
  },
)

Best Practices

  • Tách logic business ra khỏi UI widgets
  • Sử dụng const constructors khi có thể để tối ưu performance
  • Tránh rebuild không cần thiết với keys phù hợp
  • Sử dụng ListView.builder cho danh sách dài
  • Optimize images với cache và compression
  • Test widgets với widget tests

Performance Tips

  1. Sử dụng const widgets để tránh rebuild
  2. Lazy load với ListView.builder thay vì ListView
  3. Cache expensive computations với Memoization
  4. Optimize images với proper formats và sizes
  5. Profile app với Flutter DevTools
