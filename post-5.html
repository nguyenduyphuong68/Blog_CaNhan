Docker hóa Java Applications

Docker DevOps Java

Containerization đã trở nên thiết yếu cho việc triển khai Java applications hiện đại. Docker cung cấp môi trường nhất quán trên development, testing, và production.

Dockerfile Cơ Bản

Bắt đầu với một Dockerfile đơn giản cho Spring Boot application:

FROM openjdk:17-jdk-slim

WORKDIR /app

COPY target/myapp.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]

Multi-Stage Builds

Multi-stage builds giảm kích thước image bằng cách tách build và runtime environments:

# Build stage
FROM maven:3.8-openjdk-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# Runtime stage
FROM openjdk:17-jre-slim
WORKDIR /app
COPY --from=build /app/target/myapp.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

JVM Tuning cho Containers

Cấu hình JVM để tuân thủ giới hạn memory của container:

ENTRYPOINT ["java", \
    "-XX:+UseContainerSupport", \
    "-XX:MaxRAMPercentage=75.0", \
    "-jar", "app.jar"]

Best Practices

  • Sử dụng base image tags cụ thể, không dùng latest
  • Tối thiểu hóa layers và kết hợp các lệnh RUN
  • Sử dụng .dockerignore để loại trừ các files không cần thiết
  • Thiết lập memory limits phù hợp
  • Sử dụng health checks cho container orchestration
