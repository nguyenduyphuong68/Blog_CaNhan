Kỹ Thuật Testing Nâng Cao với JUnit 5

Testing JUnit Quality

JUnit 5 giới thiệu các tính năng mạnh mẽ vượt xa basic unit testing. Các kỹ thuật nâng cao này giúp viết các tests dễ bảo trì và toàn diện hơn.

Parameterized Tests

Parameterized tests cho phép chạy cùng một test với các inputs khác nhau:

@ParameterizedTest
@ValueSource(ints = {1, 3, 5, 7, 9})
void testIsOdd(int number) {
    assertTrue(number % 2 == 1);
}

@ParameterizedTest
@CsvSource({
    "1, true",
    "2, false",
    "3, true"
})
void testIsOdd(int number, boolean expected) {
    assertEquals(expected, number % 2 == 1);
}

Test Templates

Test templates cung cấp cách để định nghĩa test cases có thể được gọi nhiều lần với các configurations khác nhau:

@TestTemplate
@ExtendWith(MyTestTemplateInvocationContextProvider.class)
void testTemplate(String input, int expected) {
    assertEquals(expected, calculator.calculate(input));
}

Custom Extensions

Tạo các test extensions có thể tái sử dụng cho setup và teardown logic chung:

public class DatabaseExtension implements BeforeEachCallback, AfterEachCallback {
    
    @Override
    public void beforeEach(ExtensionContext context) {
        // Setup database
    }
    
    @Override
    public void afterEach(ExtensionContext context) {
        // Cleanup database
    }
}

@ExtendWith(DatabaseExtension.class)
class DatabaseTest {
    // Tests here
}

Dynamic Tests

Tạo tests động tại runtime:

@TestFactory
Stream<DynamicTest> dynamicTests() {
    return Stream.of("A", "B", "C")
        .map(letter -> DynamicTest.dynamicTest(
            "Test " + letter,
            () -> assertNotNull(letter)
        ));
}

Nested Tests

Tổ chức các tests liên quan bằng cách sử dụng nested test classes:

@Nested
class WhenUserIsAuthenticated {
    @Test
    void shouldAllowAccess() {
        // Test logic
    }
}

@Nested
class WhenUserIsNotAuthenticated {
    @Test
    void shouldDenyAccess() {
        // Test logic
    }
}

Best Practices

  • Sử dụng tên test mô tả rõ ràng những gì đang được test
  • Giữ tests độc lập và tách biệt
  • Sử dụng parameterized tests để giảm trùng lặp
  • Tận dụng extensions cho cross-cutting concerns
  • Viết tests dễ hiểu và dễ bảo trì
