Xây Dựng RESTful APIs với Spring WebFlux

Spring Reactive API

Spring WebFlux cung cấp mô hình reactive programming để xây dựng web applications. Nó được xây dựng trên Project Reactor và hỗ trợ non-blocking, event-driven programming.

Tại Sao Sử Dụng WebFlux?

WebFlux lý tưởng cho các applications cần xử lý nhiều concurrent connections với số lượng threads tối thiểu. Nó sử dụng một số lượng nhỏ threads để xử lý nhiều requests thông qua non-blocking I/O.

Reactive Controllers

WebFlux controllers trả về các reactive types như Mono và Flux:

@RestController
@RequestMapping("/api/products")
public class ProductController {
    
    @Autowired
    private ProductService productService;
    
    @GetMapping("/{id}")
    public Mono<Product> getProduct(@PathVariable String id) {
        return productService.findById(id);
    }
    
    @GetMapping
    public Flux<Product> getAllProducts() {
        return productService.findAll();
    }
}

Reactive Service Layer

Services trả về các reactive types có thể compose dễ dàng:

@Service
public class ProductService {
    
    public Mono<Product> findById(String id) {
        return repository.findById(id)
            .switchIfEmpty(Mono.error(new ProductNotFoundException()));
    }
    
    public Flux<Product> findAll() {
        return repository.findAll()
            .filter(Product::isActive)
            .take(100);
    }
}

Xử Lý Lỗi (Error Handling)

WebFlux cung cấp error handling thanh lịch với các reactive operators:

@ExceptionHandler(ProductNotFoundException.class)
public Mono<ResponseEntity<ErrorResponse>> handleNotFound(
    ProductNotFoundException ex) {
    return Mono.just(ResponseEntity
        .status(HttpStatus.NOT_FOUND)
        .body(new ErrorResponse(ex.getMessage())));
}

Khi Nào Sử Dụng WebFlux

  • Yêu cầu high concurrency
  • Non-blocking I/O operations
  • Reactive data streams
  • Microservices với nhu cầu high throughput
