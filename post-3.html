Bài 3: Lập trình đa tuyến (Multithreading)

Java Threading Concurrency Multithreading

Đây là kỹ thuật quan trọng để ứng dụng mạng có thể xử lý nhiều yêu cầu cùng lúc.

Khái niệm

Thread (tuyến) là dòng điều khiển tuần tự bên trong một chương trình. Một chương trình có thể có nhiều thread chạy đồng thời, mỗi thread thực hiện một nhiệm vụ riêng biệt.

Lý do sử dụng

  • Tăng tốc độ tương tác: Cho phép ứng dụng phản hồi nhanh hơn bằng cách xử lý nhiều tác vụ đồng thời
  • Dễ dàng giao tiếp: Các thread có thể chia sẻ bộ nhớ, giúp trao đổi dữ liệu hiệu quả
  • Tận dụng đa lõi: Sử dụng tối đa sức mạnh của CPU đa lõi
  • Ứng dụng mạng: Cần thiết cho server xử lý nhiều client đồng thời

Cách tạo Thread trong Java

Phương pháp 1: Kế thừa lớp Thread

Kế thừa lớp Thread và ghi đè phương thức run():

class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Thread: " + i);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

// Sử dụng
MyThread thread = new MyThread();
thread.start(); // Bắt đầu thread

Phương pháp 2: Triển khai giao diện Runnable

Triển khai giao diện Runnable và truyền vào constructor của Thread:

class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Runnable: " + i);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

// Sử dụng
Thread thread = new Thread(new MyRunnable());
thread.start();

// Hoặc sử dụng lambda expression
Thread thread2 = new Thread(() -> {
    System.out.println("Thread với lambda");
});
thread2.start();

Quản lý Thread

Độ ưu tiên (Priority)

Có thể điều chỉnh độ ưu tiên từ mức 1 đến 10 (mặc định là 5):

Thread thread = new Thread(new MyRunnable());
thread.setPriority(Thread.MAX_PRIORITY); // 10
// hoặc
thread.setPriority(Thread.MIN_PRIORITY); // 1
thread.setPriority(Thread.NORM_PRIORITY); // 5

Thực thi và tạm dừng

  • start(): Bắt đầu thực thi thread
  • sleep(milliseconds): Tạm dừng thread trong một khoảng thời gian
  • yield(): Nhường CPU cho các thread khác có cùng độ ưu tiên
  • join(): Đợi thread kết thúc trước khi tiếp tục
Thread thread1 = new Thread(() -> {
    System.out.println("Thread 1 đang chạy");
});

Thread thread2 = new Thread(() -> {
    try {
        thread1.join(); // Đợi thread1 kết thúc
        System.out.println("Thread 2 sau khi thread1 kết thúc");
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
});

thread1.start();
thread2.start();

Đồng bộ hóa (Synchronization)

Sử dụng từ khóa synchronized để tránh xung đột dữ liệu khi nhiều thread cùng truy cập một vùng nhớ:

Synchronized Method

class Counter {
    private int count = 0;
    
    public synchronized void increment() {
        count++;
    }
    
    public synchronized int getCount() {
        return count;
    }
}

Synchronized Block

class Counter {
    private int count = 0;
    private Object lock = new Object();
    
    public void increment() {
        synchronized (lock) {
            count++;
        }
    }
}

Vấn đề Race Condition

Khi nhiều thread cùng truy cập và thay đổi dữ liệu chung mà không có đồng bộ hóa, có thể xảy ra race condition:

// Vấn đề: Không đồng bộ hóa
class UnsafeCounter {
    private int count = 0;
    
    public void increment() {
        count++; // Không an toàn với nhiều thread
    }
}

// Giải pháp: Sử dụng synchronized
class SafeCounter {
    private int count = 0;
    
    public synchronized void increment() {
        count++; // An toàn với nhiều thread
    }
}

Best Practices

  • Ưu tiên sử dụng Runnable thay vì kế thừa Thread
  • Luôn sử dụng synchronized khi nhiều thread truy cập dữ liệu chung
  • Tránh deadlock bằng cách không giữ nhiều lock cùng lúc
  • Sử dụng Thread.sleep() một cách hợp lý, không lạm dụng
  • Xem xét sử dụng java.util.concurrent package cho các tác vụ phức tạp hơn
