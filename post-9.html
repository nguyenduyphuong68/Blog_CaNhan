Bài 9: Phân tán đối tượng bằng Java RMI

Java RMI Distributed Systems Remote Objects

Công nghệ cho phép gọi phương thức của một đối tượng ở máy xa như đang gọi cục bộ.

Thành phần kiến trúc

Remote Interface

Giao diện định nghĩa các phương thức gọi từ xa. Phải kế thừa từ java.rmi.Remote và tất cả các phương thức phải khai báo throws RemoteException:

import java.rmi.Remote;
import java.rmi.RemoteException;

public interface Calculator extends Remote {
    int add(int a, int b) throws RemoteException;
    int subtract(int a, int b) throws RemoteException;
    int multiply(int a, int b) throws RemoteException;
    double divide(int a, int b) throws RemoteException;
}

Stubs & Skeletons

  • Stubs: Các đại diện phía Client để xử lý giao tiếp trung gian. Stub được tạo tự động và chạy trên client.
  • Skeletons: Các đại diện phía Server (từ Java 2 trở đi, skeleton không còn cần thiết, RMI runtime xử lý trực tiếp).

Registry

Dịch vụ đăng ký và tìm kiếm đối tượng từ xa. RMI Registry chạy trên một port (mặc định 1099) và lưu trữ mapping giữa tên và remote object reference.

Các bước cài đặt

Bước 1: Tạo Remote Interface

Tạo Remote Interface kế thừa Remote:

import java.rmi.Remote;
import java.rmi.RemoteException;

public interface HelloService extends Remote {
    String sayHello(String name) throws RemoteException;
}

Bước 2: Triển khai Interface tại Server

Triển khai Interface và kế thừa UnicastRemoteObject:

import java.rmi.RemoteException;
import java.rmi.server.UnicastRemoteObject;

public class HelloServiceImpl extends UnicastRemoteObject 
        implements HelloService {
    
    public HelloServiceImpl() throws RemoteException {
        super();
    }
    
    @Override
    public String sayHello(String name) throws RemoteException {
        return "Hello, " + name + "!";
    }
}

Bước 3: Đăng ký đối tượng với RMI Registry

Đăng ký đối tượng với RMI Registry bằng Naming.bind():

import java.rmi.Naming;
import java.rmi.registry.LocateRegistry;

public class HelloServer {
    public static void main(String[] args) {
        try {
            // Tạo registry trên port 1099
            LocateRegistry.createRegistry(1099);
            
            // Tạo remote object
            HelloService service = new HelloServiceImpl();
            
            // Đăng ký với tên "HelloService"
            Naming.bind("rmi://localhost:1099/HelloService", service);
            
            System.out.println("Server đã sẵn sàng...");
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Bước 4: Client tìm đối tượng và gọi phương thức

Client tìm đối tượng bằng Naming.lookup() và gọi phương thức:

import java.rmi.Naming;

public class HelloClient {
    public static void main(String[] args) {
        try {
            // Tìm remote object
            HelloService service = (HelloService) 
                Naming.lookup("rmi://localhost:1099/HelloService");
            
            // Gọi phương thức từ xa
            String response = service.sayHello("Java RMI");
            System.out.println("Response: " + response);
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Ví dụ hoàn chỉnh: Calculator Service

Remote Interface

import java.rmi.Remote;
import java.rmi.RemoteException;

public interface Calculator extends Remote {
    int add(int a, int b) throws RemoteException;
    int subtract(int a, int b) throws RemoteException;
    int multiply(int a, int b) throws RemoteException;
    double divide(int a, int b) throws RemoteException;
}

Implementation

import java.rmi.RemoteException;
import java.rmi.server.UnicastRemoteObject;

public class CalculatorImpl extends UnicastRemoteObject 
        implements Calculator {
    
    public CalculatorImpl() throws RemoteException {
        super();
    }
    
    @Override
    public int add(int a, int b) throws RemoteException {
        return a + b;
    }
    
    @Override
    public int subtract(int a, int b) throws RemoteException {
        return a - b;
    }
    
    @Override
    public int multiply(int a, int b) throws RemoteException {
        return a * b;
    }
    
    @Override
    public double divide(int a, int b) throws RemoteException {
        if (b == 0) {
            throw new RemoteException("Không thể chia cho 0");
        }
        return (double) a / b;
    }
}

Server

import java.rmi.Naming;
import java.rmi.registry.LocateRegistry;

public class CalculatorServer {
    public static void main(String[] args) {
        try {
            LocateRegistry.createRegistry(1099);
            
            Calculator calculator = new CalculatorImpl();
            Naming.bind("rmi://localhost:1099/Calculator", calculator);
            
            System.out.println("Calculator Server đã sẵn sàng...");
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Client

import java.rmi.Naming;

public class CalculatorClient {
    public static void main(String[] args) {
        try {
            Calculator calc = (Calculator) 
                Naming.lookup("rmi://localhost:1099/Calculator");
            
            System.out.println("10 + 5 = " + calc.add(10, 5));
            System.out.println("10 - 5 = " + calc.subtract(10, 5));
            System.out.println("10 * 5 = " + calc.multiply(10, 5));
            System.out.println("10 / 5 = " + calc.divide(10, 5));
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Chạy ứng dụng RMI

  1. Compile các file: 
    javac *.java
  2. Generate Stub (Java 8 trở xuống): 
    rmic CalculatorImpl
  3. Chạy RMI Registry (nếu chưa chạy): 
    start rmiregistry
  4. Chạy Server: 
    java CalculatorServer
  5. Chạy Client (trong terminal khác): 
    java CalculatorClient

Ưu điểm và nhược điểm

Ưu điểm

  • Đơn giản, dễ sử dụng với Java
  • Tự động xử lý serialization/deserialization
  • Hỗ trợ garbage collection cho remote objects
  • Tích hợp tốt với Java

Nhược điểm

  • Chỉ hoạt động với Java (không cross-platform)
  • Bảo mật có thể là vấn đề (cần cấu hình security policy)
  • Hiệu suất có thể chậm hơn so với các giải pháp khác
  • Firewall có thể chặn RMI traffic

Best Practices

  • Luôn khai báo throws RemoteException cho các phương thức remote
  • Sử dụng security policy phù hợp khi chạy RMI
  • Xử lý exceptions một cách phù hợp
  • Xem xét sử dụng RMI over SSL/TLS cho bảo mật
  • Đóng registry và unexport objects khi không còn cần thiết
  • Xem xét các giải pháp thay thế như REST, gRPC cho các ứng dụng cross-platform
